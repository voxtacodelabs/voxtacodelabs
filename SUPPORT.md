# Voxta CodeLabs Tecnologia Ltda.

---

# Bem-vindo

Obrigado por utilizar um projeto da **Voxta CodeLabs Tecnologia Ltda.**

Nosso objetivo é construir soluções inteligentes, confiáveis e bem documentadas. Caso tenha dúvidas, sugestões ou encontre algum problema, este documento explica como obter ajuda.

---

# Antes de solicitar suporte

Antes de abrir uma solicitação, recomendamos que você:

- Leia o README.md
- Consulte a documentação do projeto
- Verifique as Issues existentes
- Consulte as Discussions
- Confira o CHANGELOG
- Verifique o ROADMAP

Em muitos casos, sua dúvida já pode ter sido respondida.

---

# Tipos de Solicitação

## 💬 Dúvidas

Caso tenha dúvidas sobre utilização do projeto:

- Utilize GitHub Discussions (quando disponível)
- Consulte a documentação oficial
- Verifique exemplos disponíveis

---

## 🐞 Bugs

Encontrou um erro?

Abra uma Issue contendo:

- descrição clara;
- passos para reprodução;
- comportamento esperado;
- comportamento observado;
- versão utilizada;
- sistema operacional;
- evidências (prints, logs ou vídeos).

---

## ✨ Novas Funcionalidades

Sugestões são sempre bem-vindas.

Antes de abrir uma solicitação:

- verifique se já existe uma Feature Request semelhante;
- descreva claramente o problema que deseja resolver;
- explique os benefícios esperados.

---

## 📚 Documentação

Encontrou erros ou deseja melhorar a documentação?

Abra uma Issue ou envie um Pull Request seguindo as diretrizes do arquivo `CONTRIBUTING.md`.

---

# Segurança

Para vulnerabilidades de segurança:

❌ Não abra uma Issue pública.

Consulte o documento:

```
SECURITY.md
```

e siga o processo de divulgação responsável.

---

# Contribuições

Interessado em contribuir?

Leia primeiro:

- CONTRIBUTING.md
- CODE_OF_CONDUCT.md

Esses documentos explicam os padrões adotados pela Voxta CodeLabs.

---

# Tempo de Resposta

Embora façamos o possível para responder rapidamente, o tempo pode variar conforme:

- complexidade da solicitação;
- disponibilidade dos mantenedores;
- prioridade do projeto.

Solicitações relacionadas à segurança possuem prioridade.

---

# O que inclui suporte

A equipe poderá auxiliar com:

- instalação;
- configuração;
- utilização das funcionalidades;
- dúvidas sobre documentação;
- esclarecimentos sobre arquitetura;
- problemas identificados no projeto.

---

# O que não inclui

O suporte normalmente não cobre:

- customizações específicas;
- desenvolvimento sob demanda;
- consultoria individual;
- problemas causados por modificações de terceiros;
- suporte a versões descontinuadas.

---

# Nossa Filosofia

Na Voxta CodeLabs acreditamos que um bom software é acompanhado de uma excelente documentação e de uma comunidade colaborativa.

Toda interação deve ocorrer com respeito, cordialidade e espírito de colaboração.
